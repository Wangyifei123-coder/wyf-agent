# PLANS.md — 计划生命周期

## 计划状态

- `draft` — 草案，待评审
- `active` — 当前正在执行（放在 `docs/exec-plans/active/`）
- `blocked` — 有依赖未解决
- `completed` — 已完成并验证（移到 `docs/exec-plans/completed/`）
- `abandoned` — 决定不做

## 规则

- 同一时间最多一个 `active` 计划
- 计划必须包含：目标、验收标准、预估步骤、风险
- 完成后必须有可运行证据
- 计划文件用日期前缀命名：`2026-06-13-initial-setup.md`
