# DESIGN.md — 设计决策

## 核心信念

1. **Prompt 即代码** — Prompt 纳入版本管理、代码审查、回归测试
2. **工具即契约** — 工具 schema 是接口契约，变更需向后兼容
3. **失败是常态** — 每一步都有 fallback：重试 → 降级 → 人工兜底
4. **可观测优先** — 没有 trace 的 Agent 就是黑盒
5. **最小权限** — 工具只给必要的权限
6. **人在回路** — 关键决策需人工确认

## 技术选型理由

| 决策 | 选型 | 理由 |
|------|------|------|
| LLM 网关 | LiteLLM | 统一接口，支持 100+ 模型 |
| 向量存储 | ChromaDB | 轻量级，本地部署，Python 原生 |
| 推理框架 | 自建 ReAct | 完全可控，不引入重依赖 |
| Web 框架 | FastAPI | 异步原生，自动 OpenAPI 文档 |
| 日志 | structlog | 结构化 JSON，生产级 |
| 追踪 | OpenTelemetry | 行业标准，可对接多种后端 |
