# 多模态图片支持设计文档

**日期**: 2026-06-15
**状态**: 已批准
**范围**: 模型升级 + 图片输入 + 图片入库

---

## 1. 目标

- 模型从 `mimo-v2.5-pro` 升级到 `mimo-v2.5`（支持多模态）
- 前端支持图片上传和粘贴
- 对话时图片直接传给 LLM 分析
- 图片入库采用 LLM 描述方案（存文本，不存图片向量）

## 2. 架构变更

### 2.1 模型配置

```
LLM_PRIMARY_MODEL=anthropic/mimo-v2.5
LLM_FALLBACK_MODEL=anthropic/mimo-v2.5
```

### 2.2 消息格式

**无图片（保持原样）：**
```json
{"role": "user", "content": "Python有什么特点？"}
```

**有图片（多模态格式）：**
```json
{
  "role": "user",
  "content": [
    {"type": "text", "text": "这张图是什么？"},
    {"type": "image_url", "image_url": {"url": "data:image/png;base64,..."}}
  ]
}
```

### 2.3 数据流

```
┌─────────────────────────────────────────────────────────────────┐
│ 前端 (Streamlit)                                                │
│  - 图片上传 (st.file_uploader)                                  │
│  - 粘贴支持 (JS clipboard API → Base64)                        │
│  - 图片预览                                                     │
└───────────────────────────┬─────────────────────────────────────┘
                            ↓
┌───────────────────────────┴─────────────────────────────────────┐
│ API 层 (FastAPI)                                                │
│  - ChatRequest: message + images (Base64 列表)                  │
└───────────────────────────┬─────────────────────────────────────┘
                            ↓
┌───────────────────────────┴─────────────────────────────────────┐
│ Agentic RAG (LangGraph)                                         │
│  - route_intent: 有图片 → doc_analysis                          │
│  - generate: 多模态消息格式传给 LLM                             │
└───────────────────────────┬─────────────────────────────────────┘
                            ↓
┌───────────────────────────┴─────────────────────────────────────┐
│ LLM Gateway (litellm)                                          │
│  - mimo-v2.5 多模态理解                                         │
└─────────────────────────────────────────────────────────────────┘
```

### 2.4 图片入库流程

```
图片上传 → LLM 生成描述 → 描述文本分块 → 存入 ChromaDB
                                              ↓
                                      可被后续问题检索
```

## 3. 涉及文件

| 文件 | 改动内容 |
|------|---------|
| `config/.env.example` | 模型名改为 mimo-v2.5 |
| `config/config.yaml` | 模型名改为 mimo-v2.5 |
| `src/gateway/client.py` | 默认模型名 |
| `src/rag/graph.py` | RAGState 加 images 字段，路由/生成支持多模态 |
| `src/api.py` | ChatRequest 加 images 字段，传递给 graph |
| `src/rag/loader.py` | 新增 load_image() 生成描述 |
| `frontend.py` | 图片上传/粘贴组件 |

## 4. 验收标准

1. 模型配置更新后，对话正常工作
2. 前端可以上传图片，支持粘贴
3. 发送带图片的问题，LLM 能正确分析图片内容
4. 图片入库后，后续问题能检索到图片相关内容
5. 现有 57 个测试继续通过
6. ruff check 和 mypy 通过

## 5. 风险

- mimo-v2.5 API 兼容性需验证
- 图片 Base64 传输大小限制
- LLM 生成图片描述的质量
