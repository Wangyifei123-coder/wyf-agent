# Prompt Calibration Guide

根指令应该写到什么程度才合适。

## 原则

1. **短入口，深链接** — AGENTS.md 保持简短，深层规则拆到 docs/
2. **机械约束优先** — 能用脚本检查的不要靠文字描述
3. **具体优于抽象** — "运行 pytest" 比 "运行测试" 好
4. **示例优于规则** — 给一个正确示例比写三条规则有效

## 校准流程

1. 写初版 prompt
2. 让 agent 执行一个任务
3. 观察 agent 在哪里偏离预期
4. 在偏离处添加具体约束
5. 重复 2-4，直到 agent 行为稳定

## 过度约束的信号

- Agent 频繁"读完所有规则后什么都不做"
- Agent 花大量时间在规则解释上
- Agent 开始"创造性地解读"规则来规避

## 约束不足的信号

- Agent 跳过验证步骤
- Agent 超出范围做无关改动
- Agent 提前宣布完成
