# RAG 技术介绍

RAG（Retrieval-Augmented Generation，检索增强生成）是一种结合信息检索和文本生成的 AI 技术。

## 核心原理

RAG 的工作流程分为三个阶段：

1. **检索（Retrieval）**：根据用户查询，从知识库中检索相关文档
2. **增强（Augmented）**：将检索到的文档作为上下文，与用户问题一起输入给大语言模型
3. **生成（Generation）**：大语言模型基于上下文生成回答

## 优势

- **减少幻觉**：基于真实文档生成回答，降低模型编造信息的概率
- **知识可更新**：只需更新知识库，无需重新训练模型
- **可追溯性**：回答可以标注引用来源，便于验证

## 技术选型

| 组件 | 常用方案 |
|------|---------|
| 向量数据库 | ChromaDB、Milvus、Qdrant、Pinecone |
| Embedding 模型 | text2vec、BGE、OpenAI Embedding |
| 文档分块 | 递归字符切分、语义切分、结构感知切分 |
| 重排序 | RRF、Cross-Encoder、ColBERT |

## Agentic RAG

Agentic RAG 在传统 RAG 基础上引入 Agent 能力：

- **意图路由**：判断是否需要检索
- **查询改写**：将口语化问题转为专业检索语句
- **自我评估**：判断检索结果是否足够，不够则迭代优化
- **问题分解**：将复杂问题拆分为子问题分别检索
