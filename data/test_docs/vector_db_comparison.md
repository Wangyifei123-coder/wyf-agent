# 向量数据库对比

向量数据库是专门为存储和检索高维向量设计的数据库，广泛应用于 AI 和机器学习领域。

## 主流向量数据库

### ChromaDB
- 类型：嵌入式
- 语言：Python
- 特点：轻量、零配置、适合原型开发
- 持久化：支持
- 适用场景：小到中等规模项目

### Milvus
- 类型：分布式服务
- 语言：Go + C++
- 特点：高性能、水平扩展、生产级
- 持久化：支持
- 适用场景：大规模生产环境

### Qdrant
- 类型：服务/嵌入式
- 语言：Rust
- 特点：高性能、支持过滤搜索
- 持久化：支持
- 适用场景：需要高性能的场景

### Pinecone
- 类型：云服务
- 特点：全托管、零运维
- 持久化：云端
- 适用场景：不想自建基础设施

### Weaviate
- 类型：服务
- 语言：Go
- 特点：内置向量化、GraphQL API
- 持久化：支持
- 适用场景：需要混合搜索

## 选型建议

| 场景 | 推荐方案 |
|------|---------|
| 原型开发 | ChromaDB |
| 中等规模 | Qdrant |
| 大规模生产 | Milvus |
| 全托管 | Pinecone |
| 已有 PostgreSQL | pgvector |

## 性能对比

| 数据库 | 10万向量查询延迟 | 内存占用 | 扩展性 |
|--------|-----------------|---------|--------|
| ChromaDB | ~10ms | 低 | 单机 |
| Milvus | ~5ms | 中 | 分布式 |
| Qdrant | ~3ms | 低 | 分布式 |
| Pinecone | ~8ms | 未知 | 自动 |
